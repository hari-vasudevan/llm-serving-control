# LLM Prompt Library — Chapter 3 (Ollama / qwen2.5:7b)
#
# Purpose:
#   These prompts are queued by the MATLAB load generator each tick.
#   They are chosen to produce realistic, variable-length responses
#   so that inference latency is meaningfully non-constant — giving
#   the cascade controller real dynamics to regulate.
#
# Design criteria:
#   - SHORT prompts  (~10-30 token response):  fast path, low latency
#   - MEDIUM prompts (~50-100 token response): mid-range latency
#   - LONG prompts   (~150-250 token response): slow path, stress test
#
#   The load generator samples uniformly from this list each tick.
#   Mixing categories creates the latency variance the p95 metric
#   is sensitive to — exactly what the outer loop must regulate.
#
# Format: one prompt per line, category tag in brackets at the start.
# Lines beginning with # are comments and are ignored by the loader.
#
